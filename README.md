# CrossLingual-SummaryEval
An Analysis of Summarization Quality, Repetition, and Model Similarity in English and Hindi Using XL-Sum

Text summarization is the task of converting a long text into a shorter version while keeping the main meaning and important information. Large language models (LLMs) are now widely used for summarization, but their performance is still not fully studied across different languages, especially in lower-resource languages such as Hindi. In this project, we evaluate two models, GPT-4o mini and Sarvam-30B, on English and Hindi news articles from the XL-Sum dataset.

For the experiment, we selected 1,000 English samples and 1,000 Hindi samples. Each model was tested on both languages. We used two types of tasks. The first was a close-ended summarization task, where the model generated a summary of the news article. The second was an open-ended generation task, where the model generated a social media headline, explained the significance of the news, and identified the nearest event or related event that the article reminded it of. For each task, we performed three runs to study the consistency and variation in the model outputs.

We evaluate the generated summaries using different methods. Lexical and semantic similarity are measured using ROUGE-1, ROUGE-2, and BERTScore. Factual correctness and information coverage are evaluated using a QA-based approach inspired by QAGS and QuestEval. Hallucinations and unsupported claims are checked using an NLI-based claim verification method inspired by FENICE. In addition to summary quality, we also study whether the models produce diverse outputs or whether their answers become similar across runs and across models. For this, we analyze intra-model repetition and inter-model homogeneity using sentence embeddings, named entity overlap, and structural patterns in the generated outputs.

This project provides a direct comparison between GPT-4o mini and Sarvam-30B for English and Hindi summarization and open-ended news generation. The results help us understand the quality, factuality, consistency, diversity, and limitations of these models in multilingual news summarization. The codebase for this project is publicly available at: \url{https://github.com/Varchita-Beena/CrossLingual-SummaryEval/tree/main}

This repository contains scripts for evaluating and analyzing multilingual summary generation across OpenAI and Sarvam models. The codebase covers three main areas: (1) **LLM-based summary evaluation** through the proposed **LLM-SQuARE** framework, which measures factual consistency, summary quality, and source coverage using question-answer based evaluation; (2) **homogeneity and consistency analysis** for auxiliary generations such as social media headlines, significance explanations, and nearest-event analogies, including both intra-model and inter-model comparisons; and (3) **supporting extraction utilities** for claims and canonical event labels, which make downstream factuality and analogy analysis more interpretable. Together, these scripts provide an end-to-end toolkit for studying not only the quality of generated summaries, but also the stability, diversity, and factual grounding of related generative tasks in both English and Hindi.

- run_summary_eval_pipeline.py: Main LLM-SQuARE pipeline for summary evaluation. Generates evaluation questions, answers them from the required comparison text, judges answer consistency, saves article-level outputs, and produces aggregate analysis for factual consistency, summary quality, and source coverage.
- analyze_social_media_homogeneity.py: Analyzes intra-model and inter-model homogeneity for social media headline generation using embeddings, cosine similarity, lexical overlap, centroid analysis, and length variation.
- plot_social_media_homogeneity.py: Creates boxplots and histograms from the social media homogeneity outputs for English and Hindi, covering both intra-model and inter-model comparisons.
- analyze_significance_homogeneity.py: Analyzes intra-model and inter-model homogeneity for significance-generation outputs using embeddings, cosine similarity, lexical overlap, centroid analysis, and variation measures.
- plot_significance_homogeneity.py: Creates boxplots and histograms from the significance homogeneity outputs for English and Hindi.
analyze_nearest_text_consistency.py: Performs text-based consistency analysis for the nearest-event task without embeddings, using exact match, token overlap, normalized event keys, n-gram overlap, heuristic entity overlap, category consistency, and length variation.
- plot_nearest_text_consistency.py: Creates boxplots and histograms from the nearest-event text consistency outputs for intra-model and inter-model comparisons.
- extract_nearest_event_labels_openai.py: Uses gpt-4o-mini to extract canonical nearest-event labels, broader historical patterns, event types, years, and regions from nearest-event responses.
- analyze_nearest_event_labels.py: Compares extracted canonical nearest-event labels across models for English and Hindi run-1 outputs, measuring event-level and category-level agreement.
- extract_claims_openai.py: Extracts atomic factual claims from generated summaries using OpenAI and saves structured claim lists for downstream factuality analysis.
